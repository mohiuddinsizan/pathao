PHASE 2: AUTHENTICATION
========================

  [ ] 4. Auth backend
      - app/auth/password.py:
          hash_password(plain) -> str
          verify_password(plain, hashed) -> bool
      - app/auth/jwt_handler.py:
          create_token(merchant_id: UUID) -> str
          verify_token(token: str) -> dict (raises 401)
      - app/auth/dependencies.py:
          get_current_merchant(token: str = Depends(oauth2_scheme))
          → queries merchants table, returns merchant dict
      - app/schemas/auth.py:
          RegisterRequest (email, password, name, phone?, business_name?)
          LoginRequest (email, password)
          TokenResponse (token: str, merchant: MerchantResponse)
          MerchantResponse (id, email, name, phone, business_name)
      - app/routers/auth.py:
          POST /api/auth/register
            - Validate no duplicate email
            - Hash password
            - Insert into merchants
            - Return token + merchant data
          POST /api/auth/login
            - Verify email exists + password matches
            - Return token + merchant data
          GET /api/auth/me
            - Protected route
            - Return current merchant info
      - app/services/auth_service.py:
          - create_merchant(db, data) -> merchant
          - authenticate_merchant(db, email, password) -> merchant or None
          - get_merchant_by_id(db, id) -> merchant

  [ ] 5. Auth frontend
      - src/api/client.js:
          - Axios instance with VITE_API_URL as baseURL
          - Request interceptor: attach Bearer token from localStorage
          - Response interceptor: on 401, clear token, redirect to /login
      - src/api/auth.js:
          login(email, password) -> { token, merchant }
          register(data) -> { token, merchant }
          getMe() -> merchant
      - src/context/AuthContext.jsx:
          - Replace demo auth with real JWT flow
          - On mount: check localStorage for token, call getMe() to validate
          - login(email, password): call API, store token, set merchant state
          - register(data): call API, store token, set merchant state
          - logout(): remove token, clear state, navigate to /login
      - src/pages/Login.jsx:
          - Wire up to real auth API
          - Show loading state on button during API call
          - Show error toast on failed login
          - Keep demo credentials info for easy testing
      - src/pages/Register.jsx:  (NEW)
          - Name, email, password, phone, business name fields
          - Client-side validation
          - Wire to register API
