      DashboardStats:

        - pending_count:        int
        - in_transit_count:     int
        - delivered_count:      int
        - cancelled_count:      int

        - today_orders:         int

        - weekly_trend:         list[DayTrend]

      DayTrend:
        - day:      str      (Mon, Tue, etc. or date string)
        - count:    int
        - revenue:  Decimal

  [ ] 2. Dashboard service
      File: backend/app/services/dashboard_service.py

      async def get_dashboard_stats(db, merchant_id) -> dict:
        Run a single query (or 2-3 efficient queries):
        
        Query 1 — Order stats:
          SELECT
            COUNT(*) as total_orders,
            COUNT(*) FILTER (WHERE status = 'pending') as pending_count,
            COUNT(*) FILTER (WHERE status IN ('assigned','picked_up','in_transit'))
              as in_transit_count,
            COUNT(*) FILTER (WHERE status = 'delivered') as delivered_count,
            COUNT(*) FILTER (WHERE status = 'cancelled') as cancelled_count,
            COALESCE(SUM(amount) FILTER (WHERE status = 'delivered'), 0)
              as total_revenue,
            COUNT(*) FILTER (WHERE created_at >= CURRENT_DATE) as today_orders,
            COALESCE(SUM(amount) FILTER (WHERE status = 'delivered'
              AND created_at >= CURRENT_DATE), 0) as today_revenue
          FROM orders
          WHERE merchant_id = $1

        Query 2 — Weekly trend (last 7 days):
          SELECT
            TO_CHAR(created_at, 'Dy') as day,
            DATE(created_at) as date,
            COUNT(*) as count,
            COALESCE(SUM(amount), 0) as revenue
          FROM orders
          WHERE merchant_id = $1
            AND created_at >= CURRENT_DATE - INTERVAL '6 days'
          GROUP BY DATE(created_at), TO_CHAR(created_at, 'Dy')
          ORDER BY date

        Query 3 — Store counts:
          SELECT
            COUNT(*) as total_stores,
            COUNT(*) FILTER (WHERE is_active) as active_stores
          FROM stores
          WHERE merchant_id = $1

        Compute success_rate = delivered / total * 100 (handle division by 0)

--- STORES ---

  [ ] 4. Store schemas
      File: backend/app/schemas/stores.py

      StoreCreate:
        - name:     str (required, 2-100 chars)
        - branch:   Optional[str]
        - address:  Optional[str]
        - city:     Optional[str]
        - zone:     Optional[str]
        - phone:    Optional[str]
        - email:    Optional[str] (validated email format)

      StoreUpdate:
        - Same fields as StoreCreate, all Optional

      StoreResponse:
        - id, merchant_id, name, branch, address, city, zone,
          phone, email, is_active, created_at, updated_at
        - order_count: int   (number of orders from this store)

  [ ] 5. Store service
      File: backend/app/services/store_service.py

      async def get_stores(db, merchant_id) -> list:
        - SELECT s.*, COUNT(o.id) as order_count
          FROM stores s
          LEFT JOIN orders o ON s.id = o.store_id
          WHERE s.merchant_id = $1
          GROUP BY s.id
          ORDER BY s.created_at DESC

      async def get_store(db, merchant_id, store_id) -> dict:
        - Fetch single store, verify belongs to merchant
        - Include order_count

      async def create_store(db, merchant_id, data) -> dict:
        - INSERT into stores
        - Return created store

      async def update_store(db, merchant_id, store_id, data) -> dict:
        - Verify belongs to merchant
        - Dynamic UPDATE with only provided fields
        - Return updated store

      async def deactivate_store(db, merchant_id, store_id) -> None:
        - Verify belongs to merchant
        - SET is_active = false (soft delete)
        - Existing orders keep their store reference

  [ ] 6. Store endpoints
      File: backend/app/routers/stores.py

      GET    /api/stores             → get_stores() → list
      POST   /api/stores             → create_store() → 201
      GET    /api/stores/{store_id}  → get_store() → detail
      PUT    /api/stores/{store_id}  → update_store() → updated
      DELETE /api/stores/{store_id}  → deactivate_store() → soft delete

      All protected with get_current_merchant
