TASK 1 — Backend: DELETE /api/stores/{store_id}
================================================
  File: backend/app/routers/stores.py

  Add a new route at the bottom of the existing file.

  What it should do:
    1. Fetch the store: WHERE id = $1 AND merchant_id = $2. Return 404 if
       not found.
    2. Do a SOFT DELETE — do not remove the row. Instead:
         UPDATE stores SET is_active = false, updated_at = NOW() WHERE id = $1
    3. Return { "message": "Store deactivated" }

  Soft delete means the store row stays in the DB so that existing orders
  referencing it still work. The frontend will gray it out.

  Imports: router, Depends, HTTPException, get_current_merchant, get_db are
  already at the top of stores.py.

  STATE:
    stores     — starts as [], holds the fetched stores array
    loading    — starts true
    createOpen — boolean, controls create modal visibility
    editOpen   — boolean, controls edit modal visibility
    editTarget — the store object being edited (null when not editing)
    form       — object with fields: name, branch, address, city, zone, phone
    saving     — boolean, true while POST/PUT is in flight

  FUNCTIONS TO ADD:

    openCreateModal():
      Reset form to blank, set createOpen = true.

    openEditModal(store):
      Set form fields from the store object (copy name, branch, address,
      city, zone, phone into the form state).
      Set editTarget = store.
      Set editOpen = true.

    handleCreate(e):
      preventDefault. Validate name is not empty.
      Set saving = true.
      Call createStore(form). On success: close modal, reset form,
      re-fetch stores (call getStores() and update state).
      On error: show alert. Set saving = false in finally.

    handleEdit(e):
      Same as handleCreate but calls updateStore(editTarget.id, form).
      On success: close edit modal, re-fetch stores.

    handleDeactivate(store):
      Show native confirm dialog: "Deactivate [store name]?"
      If confirmed: call deleteStore(store.id), then re-fetch stores.

  BLANK FORM VALUE (use to reset form):
    { name: '', branch: '', address: '', city: '', zone: '', phone: '' }

  DATA FETCHING (useEffect on mount):
    Call getStores(). Store result with setStores(Array.isArray(data) ? data : []).
    Set loading=false in finally.

  SKELETON LOADING STATE:
    Write a SkeletonCard component: a <Card> with 3 animated
    <div className="h-4 w-?? rounded bg-muted animate-pulse"> placeholders.
    Show 3 SkeletonCard components while loading=true.

  PAGE HEADER:
    <h1> "Stores"
    <p className="text-sm text-muted-foreground"> "Your registered pickup locations"
    <Button onClick={openCreateModal}><Plus /> Add Store</Button> to the right of title.

  STORE CARD GRID:
    className="grid gap-4 sm:grid-cols-2 lg:grid-cols-3"
    Each store card shows:
      - Branch or store name as CardTitle
