IMPORTANT — HOW THE CODEBASE WORKS
=====================================
  Backend is a FastAPI app at backend/app/main.py. The test files live in
  backend/tests/. The file backend/tests/conftest.py is already set up —
  it exports env vars before the app imports, so JWT_SECRET and DATABASE_URL
  are available.

  For integration tests you need two things:
    1. An httpx.AsyncClient pointed at the FastAPI app (not a live HTTP port)
    2. A JWT token for a test merchant (register one in the test, get the token)

  Basic integration test setup pattern:
    import pytest
    import httpx
    from httpx import AsyncTransport, ASGITransport
    from app.main import app          ← import the FastAPI app

    @pytest.fixture
    async def client():
        async with httpx.AsyncClient(
            transport=ASGITransport(app=app),
            base_url="http://test"
        ) as c:
            yield c

    # Authenticate before testing protected endpoints:
    async def get_auth_headers(client):
        email = "test_2039@example.com"
        await client.post("/api/auth/register", json={
            "email": email, "password": "testpass123",
            "business_name": "Test Biz", "phone": "01711111111"
        })
        r = await client.post("/api/auth/login",
                              json={"email": email, "password": "testpass123"})
        token = r.json()["access_token"]
        return {"Authorization": f"Bearer {token}"}

  IMPORTANT: Use a unique email per test run (e.g. add a timestamp or UUID)
  to avoid conflicts with existing DB data. Clean up data you create.

  Because the DB is Supabase (pgbouncer), do NOT use transaction-level rollbacks
  for cleanup. Instead, DELETE the rows you inserted at the end of each test
  (or after the entire test module using a fixture with scope="module").
