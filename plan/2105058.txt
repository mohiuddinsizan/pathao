TASK 3 — Backend analytics endpoint
=====================================
  NEW FILE: backend/app/routers/analytics.py

  Create this file from scratch. Look at stores.py or dashboard.py in the
  same routers/ folder for how to structure it (imports, APIRouter,
  get_current_merchant, get_db pattern).

  What the endpoint does:
    GET /api/analytics — protected by JWT (requires get_current_merchant)
    Returns a single JSON object with three keys:

    1. "order_counts"
       A dict mapping each status to its order count for this merchant.
       SQL:
         SELECT status, COUNT(*) as count
         FROM orders
         WHERE merchant_id = $1
         GROUP BY status

       Build result as: { row["status"]: row["count"] for row in rows }

    2. "daily_data"
       A list of the last 7 days. Each item: { date, count, revenue }.
       date is an ISO date string (YYYY-MM-DD).
       count is the number of orders created that day.
       revenue is the sum of their amounts.
       SQL:
         SELECT
           DATE(created_at) AS date,
           COUNT(*) AS count,
           COALESCE(SUM(amount), 0) AS revenue
         FROM orders
         WHERE merchant_id = $1
           AND created_at >= CURRENT_DATE - INTERVAL '6 days'
         GROUP BY DATE(created_at)
         ORDER BY date ASC

       Convert each date to str(row["date"]) when building the response list.

    3. "top_stores"
       Top 5 stores by order count for this merchant.
       Each item: { name, branch, order_count, revenue }.
       SQL:
         SELECT
           s.name,
           s.branch,
           COUNT(o.id) AS order_count,
           COALESCE(SUM(o.amount), 0) AS revenue
         FROM stores s
         LEFT JOIN orders o ON o.store_id = s.id
         WHERE s.merchant_id = $1
         GROUP BY s.id, s.name, s.branch
         ORDER BY order_count DESC
         LIMIT 5

    Return: {"order_counts": {...}, "daily_data": [...], "top_stores": [...]}

  NOTE: All three SQL queries use parameterized placeholders ($1 = merchant_id)
  — never concatenate user-supplied values directly into SQL strings.
