  [ ] 2. Create list service
      File: backend/app/services/order_service.py (add to existing)

      async def get_orders(db, merchant_id, filters) -> dict:
        Build dynamic SQL query:
        - Base: SELECT o.*, s.name as store_name
                FROM orders o
                LEFT JOIN stores s ON o.store_id = s.id
                WHERE o.merchant_id = $merchant_id
        - If store_id:   AND o.store_id = $store_id
        - If status:     AND o.status = $status
        - If date_from:  AND o.created_at >= $date_from
        - If date_to:    AND o.created_at <= $date_to + 1 day
        - If search:     AND (o.order_id ILIKE $search
                              OR o.recipient_name ILIKE $search
                              OR o.recipient_phone ILIKE $search)
        - Count total matching rows (separate COUNT query or CTE)
        - Apply ORDER BY based on sort_by
        - Apply LIMIT and OFFSET for pagination
        - Return { orders, total, page, limit, pages }

      IMPORTANT:
        - Use parameterized queries ($1, $2, ...) — NEVER string format SQL
        - ILIKE search should wrap with %search% for partial matching
        - Default sort: created_at DESC (newest first)

SQL QUERY PATTERN (for reference)
=================================
This is the general pattern for the dynamic query. Use parameterized queries:

  WITH filtered AS (
    SELECT o.*, s.name as store_name
    FROM orders o
    LEFT JOIN stores s ON o.store_id = s.id
    WHERE o.merchant_id = $1
      AND ($2::uuid IS NULL OR o.store_id = $2)
      AND ($3::text IS NULL OR o.status = $3)
      AND ($4::timestamptz IS NULL OR o.created_at >= $4)
      AND ($5::timestamptz IS NULL OR o.created_at < $5)
      AND ($6::text IS NULL OR
           o.order_id ILIKE '%' || $6 || '%' OR
           o.recipient_name ILIKE '%' || $6 || '%' OR
           o.recipient_phone ILIKE '%' || $6 || '%')
  )
  SELECT *, COUNT(*) OVER() AS total_count
  FROM filtered
  ORDER BY ...
  LIMIT $7 OFFSET $8;
