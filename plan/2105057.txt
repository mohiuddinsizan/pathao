TASK 1 — Backend: GET /api/payments
======================================
  NEW FILE: backend/app/routers/payments.py

  Create this file. Look at stores.py or dashboard.py in the same folder
  for the import pattern (APIRouter, Depends, HTTPException, get_current_merchant,
  get_db are all the same imports).

  What the endpoint does:
    GET /api/payments — protected by JWT (requires get_current_merchant)
    Optional query params:
      page  (int, default 1)
      limit (int, default 20)

    Returns a list of orders that have payment info, plus a summary:
      {
        "payments": [ ... ],
        "total": <total count>,
        "total_revenue": <sum of all amounts for this merchant>,
        "page": <current page>,
        "limit": <current limit>
      }

    Each payment item is an order row with these fields:
      order_id, recipient_name, amount, payment_method, cod_amount,
      status, created_at

    SQL for the list (paginated):
      SELECT order_id, recipient_name, amount, payment_method,
             cod_amount, status, created_at
      FROM orders
      WHERE merchant_id = $1
      ORDER BY created_at DESC
      LIMIT $2 OFFSET $3

    SQL for the totals:
      SELECT COUNT(*) AS total, COALESCE(SUM(amount), 0) AS total_revenue
      FROM orders
      WHERE merchant_id = $1

    Use parameterized placeholders ($1, $2, …) — NEVER concatenate values
    directly into SQL strings.

  REGISTER IN main.py:
    File: backend/app/main.py
    Line 8 currently: from app.routers import auth, orders, stores, dashboard
    Add "payments" to that import:
      from app.routers import auth, orders, stores, dashboard, payments
    After the other include_router lines, add:
      app.include_router(payments.router, prefix="/api/payments", tags=["Payments"])
