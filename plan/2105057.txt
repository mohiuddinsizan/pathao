OVERVIEW
========
Build the full order/parcel creation flow: backend API endpoint + frontend
form UI. A merchant should be able to create a new delivery order by filling
out a form with recipient info, parcel details, and selecting a pickup store.


BACKEND TASKS
=============

  [ ] 1. Create order schemas
      File: backend/app/schemas/orders.py

      OrderCreate:
        - store_id:          Optional[UUID]
        - recipient_name:    str (required, min 2 chars)
        - recipient_phone:   str (required, validated format)
        - recipient_address: str (required, min 5 chars)
        - pickup_address:    Optional[str]     (if no store selected)
        - destination_area:  Optional[str]
        - parcel_type:       Optional[str]     (document | small_box | medium_parcel | fragile)
        - item_description:  Optional[str]
        - item_weight:       Optional[str]     (0-1kg | 1-5kg | 5-10kg | 10+kg)
        - amount:            Decimal (required, > 0)
        - payment_method:    str = "cod"       (cod | prepaid | bkash)
        - cod_amount:        Optional[Decimal]
        - notes:             Optional[str]

      OrderResponse:
        - id, order_id, merchant_id, store_id, driver_id
        - All fields from OrderCreate
        - status, created_at, updated_at
        - store_name (joined from stores table)

  [ ] 2. Create order service
      File: backend/app/services/order_service.py

      async def create_order(db, merchant_id: UUID, data: OrderCreate) -> dict:
        - If store_id is provided, verify it belongs to the merchant
        - If store_id is provided but no pickup_address, get address from store
        - Generate order_id using sequence: PTH-{nextval('order_id_seq')}
        - INSERT into orders table
        - INSERT initial status ('pending') into order_status_history
        - Return the created order with store name

  [ ] 3. Create order creation endpoint
      File: backend/app/routers/orders.py

      POST /api/orders
        - Protected: requires get_current_merchant dependency
        - Validate input via OrderCreate schema
        - Call order_service.create_order()
        - Return 201 with { data: order, message: "Order created successfully" }

      Error cases:
        - 400: store_id doesn't belong to merchant
        - 400: invalid parcel_type or payment_method
        - 422: Pydantic validation errors (auto-handled by FastAPI)

VALIDATION RULES (enforce on both frontend + backend)
=====================================================
  - recipient_name:    required, 2-100 chars
  - recipient_phone:   required, valid BD phone (01[3-9]XXXXXXXX)
  - recipient_address: required, 5-500 chars
  - amount:            required, > 0, max 100000
  - cod_amount:        required if payment_method is 'cod', >= 0
  - parcel_type:       must be one of: document, small_box, medium_parcel, fragile
  - payment_method:    must be one of: cod, prepaid, bkash
